总体思想
	服务器与客户端之间的通信是无连接、无状态的
	服务器提供两种功能，即目录枚举功能和取指定文件指定偏移和大小的数据。
	目录和文件
		它们统一以一个url表示
		服务器上的文件是以普通的目录方式存放
		url是以”/”分隔的，因此”/”不能作为目录和文件名的一部分，为了与windows兼容”\”也不能作为目录和文件名的一部分,此外文件名不能以”.”开头
		注意，在这里没有当前目录的概念，同样的也没有相对路径的概念。所有url都是以”/”开头，如果没有那么也认为是以”/”开头，如下面两个url是相等的
		/aaa/b
		aaa/b

	命令格式
		目录枚举功能
			请求参数
				url 想要列举的目录
			响应
				url信息列表
			url信息的结构如下
				类型 (目录或文件)
				url
				下面两个如果是目录的话，值无意义
				大小
				校验码
			错误
				a)	对象不存在
				b)	内部错误
				c)	不可识别的命令
				d)	没有权限
			注意
				如果请求参数url指向的是一个文件，则返回的是包含一个且仅有一个该文件信息的url信息的url信息列表
		读数据
			请求参数
				url 想要读的文件url
				文件偏移位置
				读数据块的大小
			响应
				数据块 (它的大小与请求参数是必须是一致的)
			错误
				a)	对象不存在
				b)	内部错误
				c)	不可识别的命令
				d)	没有权限
				e)	类型错误，它是一个目录
